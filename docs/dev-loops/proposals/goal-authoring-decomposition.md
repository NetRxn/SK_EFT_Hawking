# GAP-A proposal: goal-authoring decomposition discipline

**Status:** DRAFT for `/debrief` sign-off. **Atlas-independent** (consumes the critical-path map if present).

**Mechanizes** the RCA's goal-authoring layer + the goal-scoping finding + PD-0: 5q.F was a
research-grade monolith (formalize 4-manifold Pin⁺ bordism + Kirby–Taylor from scratch) armed as
**one** `/goal` with a "NO disclosed Prop" acceptance criterion + a "disregard scale" posture — so
when the L2 keystone turned hard, **no sanctioned layer-boundary move existed** and the loop thrashed.

## What it does
A `goal-prompt` authoring step that stops a monolith being armed without a constructive layer
structure.

## Mechanism (authoring checklist → `goal-prompt-authoring.md`)
Before arming, the author answers:
1. **One layer, or a multi-layer program?** (If the critical-path audit shows >1 large layer with a
   hard keystone → multi-layer.)
2. **If multi-layer:** arm **layer-sized** goals (or stage them), each with its own
   transcript-checkable acceptance — not one monolith whose only end-state is the distant apex.
3. **Sanction the layer-boundary bank-or-grind escalation** (PD-2): the goal explicitly permits
   banking a *mechanically-discharged* tracked Prop at a layer boundary to unblock downstream (with
   user sign-off) — instead of a blanket "NO disclosed Prop." This is the constructive move 5q.F
   lacked.
4. **Pair PD-0 with the construction** — never arm a bare "disregard scale / don't stop" prohibition
   (which is what produced the thrash).

## Where it lives (goal-safe)
`goal-prompt-authoring.md` (the reference the `goal-prompt` skill loads at arm-time). Authoring-time
only; nothing competes with `/goal`. Consumes the `critical-path-audit` map when present.
