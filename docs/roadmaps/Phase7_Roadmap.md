# Phase 7: Bundle Drafting & Consolidation

## Strategic Summary

**Created:** 2026-04-30. **Trigger:** Phase 6m R6 Lean closure SHIPPED + Phase 6i Wave 7 bundle infrastructure SHIPPED. With Phase 6 substantively closed at the research-and-formalization level, the project enters its publication phase.

**Macro-goal.** Consolidate 32+ per-wave drafts in `papers/paperN_*/` into 13 publication-target bundles per `PAPER_STRATEGY.md`, draft each bundle's LaTeX end-to-end, and reach Stage-13 GREEN across all 13 bundles before any arXiv submission.

**Scope discipline (from user 2026-04-30):** *"we'll write everything up together before anything is submitted."* Phase 7 is a **drafting-ordered** phase, not the submission-ordered roll-out described in `PAPER_STRATEGY.md` §3. Submission sequencing is a Phase 8 (post-Phase-7) concern; Phase 7 produces 13 bundles each at submission-ready state.

**Existing bundle infrastructure (already shipped, Phase 6i Wave 7).** Sentence-level `bundle_destination` schema (`scripts/sentence_state.py`); per-bundle reviewer prompts (`docs/agents/claims-reviewer-bundle-prompts.md`); `validate.py --check bundle_consistency` (CHECK 21); per-bundle Stage-13 review docs at `papers/AutomatedReviews/<DATE>-bundle-stage13/<bundle>.md`; `docs/BUNDLE_READINESS_HEATMAP.md` auto-regenerated by `scripts/bundle_readiness.py`; provenance dashboard "Bundles" tab; `papers/cluster_bundle_index.json` cross-bundle cluster registry; CLAUDE.md Tier-2 references; Pipeline Invariant #14.

**What Phase 7 adds.** The *content* consolidation. Each bundle gets a `papers/<bundle>/paper_draft.tex`, `papers/<bundle>/figures/`, `papers/<bundle>/tables/`, `papers/<bundle>/sentence_state.json`, `papers/<bundle>/audit_log.jsonl`, plus per-bundle Stage-13 closure (BLOCKERs cleared on a fresh-context review, not just on the per-paper baseline aggregated by Wave 7.4).

**Existing per-paper drafts are NOT deleted.** They remain in `papers/paperN_*/` as historical / source material per `PAPER_DRAFT_MAPPING.md`. Phase 7 lifts content, preserving sentence-level provenance via `bundle_destination` tags.

**Robustness to late-arriving Phase 6 outputs.** Phase 6 is anticipated to continue past Phase 6m (i.e., Phase 6n, 6o, ... may ship during Phase 7 execution). Phase 7a sub-wave 7a.1 builds the **append-only bundle architecture** + tooling (`bundle_append.py`, `bundle_source_manifest.py`, validate.py CHECK 22 `bundle_source_freshness`) + protocol (`LATE_PHASE6_ABSORPTION_PROTOCOL.md`) before any deep-paper bundles are drafted. Subsequent Phase 7b/7c/... sub-phases consume these artifacts. The protocol classifies new Phase 6X output into branches D.1 (bundle not yet drafted; passive pickup), D.2 (additive append via tooling), D.3 (revision required; manual author judgment with documented branch), and propagates downstream to F (flagship) automatically. New Phase 6X output that doesn't fit any of the existing 13 bundles requires user authorization for 14th+ bundle target per Pipeline Invariant #14.

### Wave 0 — Pre-Flight: source-level blocker cleanup [Pipeline: Stages 7, 12, 13]

**Goal.** Bring all 32+ per-paper drafts (post-supersession) to 🟢 GREEN or 🟡 YELLOW state at the per-paper Stage-13 level, *before* lifting any content into bundles. A bundle's quality is bounded by its sources; cleaning sources first eliminates mass-rework during bundle assembly.

**Why upstream first.** Per Phase 6i Wave 7's empirical heatmap, ~85% of bundle-level blockers trace back to specific per-paper findings (CitationIntegrity 121, CountFreshness 14, ParameterProvenance 14). Resolving these *at the source paper* is more efficient than resolving them per-bundle (where the same finding may surface across 2–3 bundles).

**Sub-waves:**

#### 7.0.1 — CitationIntegrity bulk sweep [~121 findings]

- Run `scripts/back_fill_primary_sources.py --fetch` against all unmapped bibkeys; verify cache populated.
- Re-run `validate.py --check citation_primary_sources_present` until PASS for every paper.
- For findings flagging mismatched-DOI / wrong-target: human verification cycle via provenance dashboard; one batch per "owner team" assignment.
- Output: 0 CitationIntegrity blockers across all 32 papers.

**Estimated effort:** 1–2 person-weeks (mostly cache-verification + DOI confirmations).

#### 7.0.2 — CountFreshness regen [~14 findings]

- `uv run python scripts/update_counts.py` regenerates `docs/counts.json` + `docs/counts.tex`.
- `uv run python scripts/render_paper_tables.py` regenerates auto-tabulated paper tables.
- `validate.py --check counts_fresh --check tables_fresh` to PASS.

**Estimated effort:** ~1 person-day (mostly automation).

#### 7.0.3 — ParameterProvenance human verification [~14 findings]

- Provenance dashboard at `localhost:8050` walks each open ParameterProvenance entry; user / collaborator confirms primary-source extraction.
- Updates `PARAMETER_PROVENANCE` `human_verified_date` field.
- `validate.py --check parameter_provenance --strict` to PASS.

**Estimated effort:** ~1 person-week (humility verification).

#### 7.0.4 — Misc finding resolution [~55 unclassified]

- Per-paper triage of remaining "unclassified" findings.
- Most are minor advisory items (formatting, abstract length, wording); resolve in batches per paper.

**Estimated effort:** ~1 person-week.

#### 7.0.5 — Per-paper Stage-13 re-invocation

- For each paper that received fixes in 7.0.1–7.0.4, re-invoke `physics-qa:claims-reviewer` and `physics-qa:figure-reviewer` for fresh-context confirmation.
- Update `docs/review_finding_supersessions.json` with `meta.status: open → fixed` for every closed finding.
- Re-regenerate `BUNDLE_READINESS_HEATMAP.md`.

**Estimated effort:** ~1 person-week.

**Decision Gate 7.0:** at Wave 0 close, every paper in `papers/paperN_*/` is at 🟢 GREEN or 🟡 YELLOW per-paper readiness. **Bundle-level RED status may persist (those are bundle-architectural issues, not source-quality issues), but no bundle finding traces back to an unresolved source-paper finding.** The bundle-lift work in Waves 1–13 starts from a clean source baseline.
