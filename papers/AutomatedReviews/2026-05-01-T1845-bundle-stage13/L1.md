## QI Candidate

**Pattern (carryover from round-2):** Body-only fixes for narrative
drift in cross-section (abstract / body / conclusion / reproducibility)
papers. The round-2 incident demonstrated that a single Lean-theorem
correction can cascade correctly across companion papers
(L1↔paper25) but still miss internal cross-section parity within a
single paper (L1 abstract vs L1 body). Round-3 verifies this is now
clean for L1, but the QI register entry from round-2 still applies:

> Extend the `paper_lean_refs_live` validate.py check to enumerate
> **every** `\verb|*|` / `\texttt{*}` Lean-name reference in the
> entire paper file, comparing each to the live declaration set in
> the corresponding Lean module. The check should emit one finding
> per stale reference, so an abstract-only stale reference is not
> masked by a clean body section.

**New observation surfaced during round-3:** L1's `paper_draft.tex`
uses `\begin{theorem}` environments at lines 146/165/172/etc. but
the document preamble (`revtex4-2`) does not declare a
`theorem` environment via `amsthm` or `\newtheorem`. This produces
3+ `LaTeX Error: Environment theorem undefined` errors per
compile, all of which LaTeX continues past — the PDF is still
produced (3 pages) and all theorem statements still render as
plain prose. This is **pre-existing**, not introduced by the
round-2 fix, and round-2 marked the PDF state as clean. However:

- A reader-facing PDF produced from a `.tex` file with active
  LaTeX errors is a credibility risk if the errors were ever to
  cause divergence between the PDF and the source.
- The current state happens to be benign because LaTeX skips the
  `\begin{theorem}...\end{theorem}` and renders the body as prose,
  but a future template change could expose this.

**Recommended QI register entry:** Add a `validate.py` check
`paper_latex_compile_clean` that fails on any `! LaTeX Error:`
in `*/paper_draft.log`, regardless of whether the PDF was
ultimately produced. Severity: REQUIRED. Bundle-target: all
13 bundles. This catches pre-existing latent errors before they
become visible regressions.

**Severity assessment:** RECOMMENDED — not gating bundle state but
worth queuing for the next infrastructure-bundle pass (I1/I2)
where the validate.py check itself can land.
