## QI Candidate

**Pattern:** Body-only fixes for narrative drift in cross-section
(abstract / body / conclusion / reproducibility) papers.

The round-1 fix for 3.1 cascaded to paper25 (lockstep with L1) but
within L1 itself the fix was scoped to the body section only; the
abstract preview was missed even though round-1 explicitly flagged
the abstract location at line 50. Pattern: when a Lean-theorem-name
correction is needed, **all four standard sections** (abstract,
introduction, body, conclusion/reproducibility) need to be checked,
not just the canonical "section that defines the theorem."

**Recommended QI register entry:** Extend the `paper_lean_refs_live`
validate.py check proposed in round-1 QI to enumerate **every**
`\verb|*|` / `\texttt{*}` Lean-name reference in the entire paper
file (not stopping at the first hit), comparing each to the live
declaration set in the corresponding Lean module. The check should
emit one finding per stale reference, so an abstract-only stale
reference is not masked by a clean body section.

**Recurrence relevance:** This is the same root cause as round-1's
QI candidate (post-strengthening narrative drift), but the round-2
incident shows the issue applies **per-section**, not per-paper.
A paper can have correctly-updated body prose and stale abstract
prose simultaneously; the validator must check both.

Estimated impact: high — abstracts are the most-read part of any
paper, and a stale abstract reference reaches more readers than any
other section.
